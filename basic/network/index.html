# 为什么HTTP/1.1不能实现多路复用？

因为HTTP/1.1不是通过二进制传输，而是通过文本进行传输的，没有流的概念。在使用并行传输（多路复用）时，接收端在接收响应后，无法区分多个相应分别对应的请求，所以无法将多个请求结果重新进行组装，也就不能实现多路复用。
